# Redeploy checklist — audit #2 fixes (2026-06-29)

Two contract fixes need to go on-chain:
- **CRITICAL** — `ArcadeV3SwapRouter` callback drain (`_authorisedPool` guard).
- **HIGH** — `ArcadeLaunchpad` H-1 migration brick (try/catch in `_migrate`).

The router is stateless → a clean surgical redeploy (Path A). The launchpad is
stateful and `setV3Infra` is one-time, so wiring it to a new router or shipping
its H-1 fix means a full stack redeploy (Path B).

---

## Path A — redeploy the V3 router ONLY  (URGENT, do first, ~5 min)

Fixes the CRITICAL fund-drain. Low risk: the router holds no state.

```bash
cd contracts

# 1. Build the 0.7.6 V3 layer so out-v3 has the FIXED router bytecode
FOUNDRY_PROFILE=v3 forge build

# 2. !! unset the v3 profile before the script. The `v3` profile has skip=[],
#    so it would compile the broken lib/v4-core + lib/v4-periphery. The DEFAULT
#    profile's foundry.toml `skip` already excludes them. Do NOT pass --skip on
#    the CLI: it REPLACES the whole config skip list (so only what you name is
#    skipped, and v4-core then compiles and chokes). Just use the default profile.
unset FOUNDRY_PROFILE

# 3. Deploy the new router (addresses from web/public/deployments.json)
PRIVATE_KEY=0x<deployer> \
V3_FACTORY_ADDRESS=<addresses.v3Factory> \
ARC_USDC_ADDRESS=<addresses.usdc> \
LAUNCHPAD_ADDRESS=<addresses.launchpad> \
forge script script/RedeployV3Router.s.sol --rpc-url arc_testnet --broadcast
# -> logs "NEW ArcadeV3SwapRouter: 0x...."
```

Then:
1. Set **`NEXT_PUBLIC_V3_ROUTER_ADDRESS`** = the new address (Vercel env) and
   update `web/public/deployments.json` (`addresses.v3Router`).
2. Redeploy the frontend (push, or Vercel redeploy).
3. New trades auto-approve + use the FIXED router (`useApproveIfNeeded`).

⚠️ **The OLD router stays drainable for allowances users already granted it.**
On testnet that's acceptable. To be clean, tell users to **revoke** their old
approval (revoke.cash on the old router) — or, since it's testnet, ignore. The
launchpad's internal `v3Router` (creatorBuy) still points at the old router but
is NOT the drain vector (it pays from the launchpad's own funds, never a
victim's standing approval), so it can stay.

## Path B — full stack redeploy  (when convenient / before any real test push)

Ships BOTH fixes (new router + launchpad H-1) in a clean stack. **This abandons
all existing test tokens/curves** (they live on the old launchpad). Acceptable
on testnet.

```bash
cd contracts
forge build                      # 0.8 layer (launchpad etc.)
FOUNDRY_PROFILE=v3 forge build   # 0.7.6 V3 layer

PRIVATE_KEY=0x<deployer> \
ARC_USDC_ADDRESS=<addresses.usdc> \
TREASURY_ADDRESS=<treasury> \
forge script script/DeployTestnet.s.sol --rpc-url arc_testnet --broadcast
```

Then update **every** address in `web/public/deployments.json` + the matching
`NEXT_PUBLIC_*` Vercel envs from the script output, redeploy the frontend, and
re-seed test liquidity. Users re-approve across the board.

## Contract fixes to CARRY in the Path B redeploy (audits #2-#4)

The launchpad (and a multiswap redeploy) should ship ALL of these together:
- **H-1** migration brick try/catch — DONE in ArcadeLaunchpad._migrate.
- **CLANKER_V3 mispricing (HIGH)** — DONE: _migrate no longer overwrites
  sqrtPriceX96 with slot0, so the locked single-sided liquidity is anchored to
  the intended price, not an attacker pre-init (commit 6d4a8b0).
- **Migration seed below clearing price (MEDIUM, MEV)** — the inline V2 seed is
  ~30% under the curve-clearing price, so every graduation has a risk-free
  back-run. TODO: seed at (or quote toward) the clearing price, or add a tiny
  post-seed sync, before mainnet.
- **H-07 MultiSwap per-leg minOut (HIGH, still unfixed in the deployed
  contract)** — ArcadeMultiSwap enforces only a basket-wide minTotalOut; every
  internal leg routes with amountOutMinimum = 0/1-wei, so a sandwicher drains a
  single thin leg up to the whole basket budget. TODO (breaks the signature +
  the frontend MultiSwapCard + agent getMultiswapPlan): add a per-input
  minOut[] threaded into each router call, plus a caller-supplied usdcMidMin for
  the V2-via-USDC hop. Do this in the MultiSwap redeploy.

- **V3 Zap full-balance sweep (theft, audit #4)** — DONE: ArcadeV3Zap._sweep
  now forwards only this call's delta (pre-pull/pre-collect snapshot), so a
  parked balance can't be skimmed by the next caller. Ported the V2 Zap fix
  (commit ba0ffca). Ships with any V3-layer redeploy.

Note: the forged-Compounder-event HIGH was an OFF-CHAIN fix (commit 554f4ee),
already deployable with the frontend, no contract redeploy needed.

## MAINNET-ONLY hardening (NOT testnet, do at the mainnet cutover) — audit #4

These are NOT code-fix-now items; they are deploy-config / ops gates:
- **Decentralise the owner/treasury keys (HIGH).** escrowOwner, locker owner,
  launchpad owner, Compounder OWNER, V2 factory feeToSetter, V4 hook OWNER and
  treasury all default to the single deployer EOA 0x3a0Dd902. On mainnet set
  each to a Safe multisig (>=2/3) behind a Timelock for the high-blast setters
  (finalizeTrustedSignerRotation, setProtocolFeeBps, setFeeRecipient,
  setClaimTimelock, forfeitStaleClaim, adminRescue, setFeeTo/Setter). Add an
  _assertWirings check that owner != deployer on production chains.
- **Re-enable the escrow veto window (MEDIUM).** Set MIN_TIMELOCK =
  DEFAULT_TIMELOCK = 1 hours (the mainnet TODOs at escrow lines 92/99) so a
  compromised trustedSigner cannot authorize+claim in one block; keep the veto
  signer independent from the owner.
- **Move the Compounder operator off the deployer key** to a dedicated
  low-privilege keeper wallet (the deploy script already supports OPERATOR).
- **Pin Arc's CCTP V2 domain (MEDIUM).** web/lib/cctp.ts says 26, the tests
  say 12, the audit doc says TBD. Confirm Circle's canonical Arc domain and
  reconcile all three before any mainnet bridge use.
- **Deprecated-router revoke prompt (MEDIUM).** The old V3 router 0x3125f00C
  stays drainable for standing approvals; keep a hardcoded deprecated-router
  list and have the frontend prompt revoke(oldRouter, 0) if a non-zero
  allowance is detected. Testnet: a revoke notice is enough.

## V4-LAYER fixes (only if/when V4 goes live) — audit #4

- **Graduation brick (HIGH).** ArcadeHook._graduate sets status=GraduationStarted
  before the unlock-seed, and beforeAddLiquidity reverts on GraduationStarted,
  so every cap-crossing buy reverts. Fix: let the hook's OWN seed add bypass the
  guard (recognise sender==address(this) in beforeAddLiquidity), or flip status
  to Graduated only around the seed. Needs a V4 redeploy.
- **V4 hook buy/sell minOut hardcoded 0 (MEDIUM).** web v4hook page passes 0n
  minOut — add a real slippage floor from estimateOut before V4 launches.

## Frontend approval hardening (defense-in-depth, do with the next redeploy)

- **Exact-amount approvals (LOW but high-leverage).** useApproveIfNeeded /
  batchSwap / TradePanel / ClankerTradePanel still grant approve(spender,
  maxUint256); Permit2 signs maxUint160. This is what made the V3-router drain
  catastrophic. Default to exact-amount approvals + sign the actual amountIn.

## Recommendation

Do **Path A now** (kills the CRITICAL drain with minimal disruption). Schedule
**Path B** before the next serious testing round / mainnet so the launchpad H-1
fix and a clean new-router wiring land together. The remaining audit-#2 items
(degenerate-pool grief → pre-create the pair at createToken; per-IP rate limits;
twitter claim deadline vs timelock) are mainnet-hardening, fold them into Path B.
