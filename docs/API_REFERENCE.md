# API reference

> All `app/api/**/route.ts` handlers in `web/`. One central place for
> input/output spec, auth model, rate limits, and error codes so
> auditors + new contributors don't have to read each route file.

Last verified 2026-06-11. Generated by reading the live source; update
when adding or modifying a route.

---

## `/api/twitter-login`

**Method**: GET

**Purpose**: kick off Twitter OAuth 2.0 flow. Sets a signed state
cookie + redirects to twitter.com.

**Query params**:

| Name | Type | Required | Constraint |
|------|------|----------|------------|
| `token` | hex 0x-prefixed | yes | `isAddress()` |
| `slotIndex` | integer string | yes | `/^[0-3]$/` |
| `recipient` | hex 0x-prefixed | yes | `isAddress()` |

**Auth**: same-origin via `Sec-Fetch-Site: same-origin` or `none`
header. Cross-origin returns 403.

**Rate limit**: 10 / 60 s per IP (`apiGuard.rateLimit`).

**Env vars** (validated before rate limit to avoid timing oracle):
- `ARCADE_OAUTH_STATE_SECRET` (HMAC the state cookie body)
- `TWITTER_CLIENT_ID` (used as `client_id` in the authorize URL)

**Responses**:
- `307` redirect to `twitter.com/i/oauth2/authorize` with a state cookie
  set (`tw_state_<state>`, `sameSite=lax`, `domain=arcade.trading` in prod)
- `307` redirect from `www.arcade.trading` → `arcade.trading` (host
  canonicalization)
- `400 {"error": "Missing or invalid token"}` etc.
- `403 {"error": "Cross-origin OAuth init not allowed"}`
- `500` empty (server misconfigured — env missing)

## `/api/twitter-callback`

**Method**: GET

**Purpose**: complete Twitter OAuth, sign an EIP-712 Claim payload,
redirect back to `/claim` with the signed payload stashed in a
short-lived HttpOnly cookie.

**Query params**: `code`, `state` (from Twitter).

**Cookie input**: `tw_state_<state>` (HMAC'd state from the login route).

**Env vars**:
- `ARCADE_OAUTH_STATE_SECRET` (verify state HMAC, sign claim payload)
- `TWITTER_CLIENT_ID`, `TWITTER_CLIENT_SECRET` (exchange code for token)
- `ARCADE_BACKEND_PRIVATE_KEY` (sign EIP-712 Claim — MUST be the wallet
  registered as `trustedSigner` on the escrow)

**Auth**: state cookie HMAC + Sec-Fetch-Site (handled by the OAuth
redirect from Twitter being a top-level navigation; sameSite=lax cookie
flows back).

**Rate limit**: 30 per minute per state (rateLimitGlobal — bounds
residential proxy farms).

**Responses**:
- `302` redirect to `/claim` with `arcade_claim_payload` cookie set
  (120 s TTL, sameSite=strict, domain=arcade.trading in prod)
- `302` redirect to `/claim?error=<code>` on failure (see below)

**Error codes** (in query string):

| `error=` | Cause |
|----------|-------|
| `missing_params` | code or state absent |
| `invalid_state` | state pattern fail or cookie absent |
| `bad_state` | state HMAC fail |
| `server_misconfigured` | required env var missing |
| `token_exchange_failed` | Twitter rejected the code |
| `no_access_token` | Twitter didn't return a token |
| `invalid_addresses` | token or recipient malformed |

## `/api/claim/payload`

**Method**: GET

**Purpose**: one-shot read of the `arcade_claim_payload` cookie set by
`/api/twitter-callback`. Returns the EIP-712 Claim signature for the
client to submit to the escrow's `authorize()`.

**Auth**:
- same-origin via `Sec-Fetch-Site`
- custom header `x-arcade-claim: 1` (mitigates passive XSS exfil via
  `<img src=...>`)

**Cookie input**: `arcade_claim_payload` (signed JSON).

**Env vars**:
- `ARCADE_OAUTH_STATE_SECRET` (verify cookie HMAC)

**Responses**:
- `200 <payload JSON>` on success. The cookie is consumed (cleared
  with matching domain attr, audit FE-1) after success — one-shot.
- `403 {"error": "csrf_block"}` if `x-arcade-claim` header missing
- `404 {"error": "no_payload"}` if cookie absent
- `400 {"error": "bad_payload"}` if HMAC fail or unparseable JSON
- `500` empty if `ARCADE_OAUTH_STATE_SECRET` missing

**Cache**: `Cache-Control: no-store` on success.

## `/api/ens/forward`

**Method**: GET

**Purpose**: ENS name → address resolution via mainnet Registry + Resolver.

**Query params**:

| Name | Type | Required | Constraint |
|------|------|----------|------------|
| `name` | string | yes | length ≤ 253 |
| `debug` | `"1"` | optional | non-production only |

**Env vars**:
- `MAINNET_RPC` (preferred) or `NEXT_PUBLIC_MAINNET_RPC` (fallback)

**Rate limit**: 60 / 60 s per IP.

**Responses**:
- `200 {"address": "0x..." | null}` always
- With `debug=1` (dev only): adds `attempts[]`, `node`, `normalized`,
  `registry`. RPC URLs are redacted to host (`https://host`) so a
  vendor key in the path can never leak (audit API-1).

## `/api/ens/reverse`

**Method**: GET

**Purpose**: address → ENS name with forward-verify roundtrip.

**Query params**:

| Name | Type | Required | Constraint |
|------|------|----------|------------|
| `address` | hex 0x-prefixed | yes | `isAddress()` |
| `debug` | `"1"` | optional | non-production only |

**Env vars**: same as forward.

**Rate limit**: 30 / 60 s per IP (each reverse fans out to ~8 RPC reads).

**Responses**: `{"name": "<name>" | null}`; debug fields likewise redacted.

## `/api/og`

**Method**: GET

**Purpose**: Open Graph image generator for token detail pages.

**Query params**: `token` (address), `name`, `symbol`, `mcap`, etc.

**Rate limit**: per-route inline bucket (10 / 60 s per IP).

**Env vars**: none (reads from constants + on-chain).

**Responses**: `200` PNG via `next/og` ImageResponse. Edge runtime.

## `/api/og/brand`

**Method**: GET

**Purpose**: Brand splash OG image (homepage, /swap, etc.).

**Cache**: `Cache-Control: public, max-age=86400, s-maxage=86400,
stale-while-revalidate=604800` (24 h fresh + 7 d SWR).

**Notes**: Edge runtime. `force-dynamic` because we read `req.url` to
build the absolute logo URL.

## `/api/pin/file`

**Method**: POST (multipart/form-data)

**Purpose**: pin a file to IPFS via Pinata.

**Auth**:
- `Sec-Fetch-Site: same-origin` via `rejectCrossOrigin` (audit FSEC-003).
- Content-Length pre-check before parse.

**Env vars**: `PINATA_JWT` (server-only).

**Rate limit**: 5 / 60 s per IP (Pinata quota-protective).

**Responses**:
- `200 {"cid": "Qm..."}` on success
- `400` on unsupported MIME (whitelist: png/jpeg/webp/svg+xml/gif)
- `413` on oversized body
- `429` rate-limited

## `/api/pin/json`

**Method**: POST (application/json)

**Purpose**: pin a JSON blob to IPFS (used for token metadata).

Same auth model + env vars as `/api/pin/file`. Body cap is smaller (~32 kB).

## `/api/stats/snapshot`

**Method**: GET

**Purpose**: cached JSON of activity stats (tx count, unique wallets,
tokens launched, estimated USDC gas paid). Backs the `/stats` page and
the footer "USDC gas paid" counter.

**Auth**: none (public).

**Rate limit**: 5 / 60 s per IP (tightened from 30 / min after audit
v2 V2-F-04 — each cold call fans out ~700 parallel `eth_getLogs`).

**Cache**: Next.js ISR `revalidate = 300` (5 min cache key).

**Responses**: `{"txCount": number, "uniqueWallets": number, ...}`
with bigints stringified for JSON.

## `/api/telemetry`

**Method**: POST (application/json)

**Purpose**: sink for client-side telemetry (`lib/telemetry.ts`
`trackSwap` / `trackBridge` / `trackClaim` / `trackProviderTiming`).
Forwards a sampled subset to Sentry.

**Auth**:
- `Sec-Fetch-Site: same-origin` via `rejectCrossOrigin` (audit API-2).

**Body cap**: 4 096 bytes (audit API-2).

**Whitelists** (audit v2 ADVR-2):
- `level` ∈ `{info, warning, error}`
- `name` MUST start with one of `swap.`, `bridge.`, `claim.`,
  `provider.`, `ui.` (else replaced with `ui.unknown`)
- `data` keys filtered to a fixed allowlist; values coerced to
  primitive (no nested objects forwarded to Sentry)

**Env vars**: `SENTRY_DSN` (server-only). Without it the route
returns 200 OK with no forward.

**Rate limit**: 120 / 60 s per IP.

**Sampling**: error events 100 %; info / warning events sampled at
`SENTRY_INFO_SAMPLE_RATE` (default 5 %).

**Responses**: `200 {"ok": true}` always (degrades silently on Sentry
errors).

## `/api/v3-position/[id]`

**Method**: GET

**Purpose**: read a V3 NFT position's metadata (positionId → owner,
tokens, fees) for the positions UI.

**Auth**: none (data is on-chain and public).

**Rate limit**: not currently rate-limited (audit v2 LOW backlog item).

**Responses**: `200 <JSON>` or `404` if positionId doesn't exist.

## Shared utilities

### `lib/apiGuard.ts`

- `rejectCrossOrigin(req)` — `Sec-Fetch-Site: same-origin` gate.
- `rateLimit(req, key, max, windowMs)` — per-IP bucket, prunes lazily
  at 5 000 buckets cap.
- `rateLimitGlobal(key, max, windowMs)` — payload-keyed bucket
  (e.g. per OAuth state). Used by twitter-callback.

### Common HTTP error codes

| Code | Meaning here |
|------|--------------|
| 200 | Success |
| 400 | Bad input (query / body) |
| 403 | Cross-origin or CSRF gate fail |
| 404 | Resource absent |
| 413 | Body exceeds cap |
| 429 | Rate-limited (`Retry-After` header set) |
| 500 | Server config missing (env var absent) |
